# 📌 Стан Проекту (Project State)

## 1. Overview (Огляд проекту)

**Назва:** Fifteen Puzzles (Гра "П'ятнашки")
**Опис:** Мобільна гра, реалізована з кастомним UI-рендером на базі Skia. Наразі реалізована класична механіка з цифрами, але в майбутньому планується заміна цифр на фрагменти зображення (з камери або галереї телефону).
**Мова розробки та документації:** Українська.

## 2. Tech Stack (Стек технологій)

- **ОС розробника:** Windows 11
- **Пакетний менеджер:** pnpm
- **Фреймворк:** React Native (0.81.5) + Expo (~54.0.33)
- **Навігація:** @react-navigation/native-stack
- **Графіка та анімації:** @shopify/react-native-skia, react-native-reanimated, react-native-gesture-handler
- **Специфічні технології:** Підтримка кастомних шейдерів (.sksl, .glsl) через налаштований metro.config.js.

## 3. Architecture (Архітектура)

- `src/context/` — Провайдери глобального стану: `GameStateProvider` (AsyncStorage), `LayoutMetricsProvider` (масштабування UI), `FontProvider` (завантаження шрифтів для RN та Skia).
- `src/navigation/RootNavigator.tsx` — Маршрутизація (Home, Game, Settings, Statistic тощо).
- `src/ui/game/` — Ядро гри (рендер дошки, обробка жестів, математика сітки).
- `src/ui/skia/` — Кастомні UI-елементи на Skia (кнопки, плитки `TileSkin`, дошка `BoardSkin`).
- `src/theme/typography.ts` — Токени типографіки (Krona One, Mariupol).

## 4. Rendering Model (Модель рендерингу)

- React відповідає виключно за layout (компонування) та загальний стан екрану.
- Весь візуальний рендер гри (дошка, плитки, тіні, ефекти) відбувається через `<Canvas>` (Skia). Стандартні RN `<View>` використовуються лише для лейауту.
- React **не повинен** викликати повторний рендер (re-render) під час перетягування (drag) плиток.
- Ланцюжок оновлення: Gesture → Reanimated → Skia update.

## 5. Data Flow (Потік даних)

Gesture (Дія користувача)
↓
BoardGestureOverlay
↓
Reanimated logic
↓
Update board state (Оновлення стану дошки)
↓
GameBoardView
↓
Skia Canvas redraw (Перемальовка)

## 6. Game Model (Модель гри)

- **Дошка (Board):** Сітка 4x4 (15 плиток + 1 пуста клітинка).
- **Рух плиток:** Синхронний зсув. Можна пересувати як одну плитку, так і цілу групу (до 3 плиток) у межах одного рядка чи стовпця в бік пустого місця.
- **Стан дошки (Board state):** Масив із 16 елементів (`array[16]`), де `0` — пуста клітинка, `1..15` — ідентифікатори (ID) плиток.

## 7. Component Responsibilities (Відповідальність компонентів)

- **`GameBoardView`:** Рендерить Skia canvas, безпосередньо малює дошку та плитки.
- **`BoardGestureOverlay`:** Обробляє свайпи (жести), визначає дозволені напрямки та кроки руху.
- **`boardGeometry`:** Розраховує координати плиток, мапить індекси масиву на фізичні позиції на екрані.

## 8. Layout System (Система верстки)

- Усі розміри та відступи розраховуються динамічно через `LayoutMetricsProvider`.
- **Формула:** `snap(value * S)`.
- Де **S** — коефіцієнт масштабування екрану (screen scale factor).
- **Мета:** Забезпечити pixel-perfect верстку, яка виглядає ідентично на пристроях з різною роздільною здатністю.

## 9. Shader System (Система шейдерів)

- **Розташування:** `src/ui/skia/shaders/`.
- **Використання:** Для складного освітлення плиток, тіней та створення візуальної глибини матеріалів.
- **Компіляція:** Відбувається нативно через налаштований `metro.config.js`.

## 10. Performance Rules (Правила продуктивності)

- Уникати React re-render під час перетягування плиток будь-якою ціною.
- Обробка жестів повинна працювати виключно всередині `Reanimated` (у UI thread).
- Skia використовується для всього візуального рендерингу (не використовувати RN тіні чи бордери для ігрових елементів).
- Метрики верстки (Layout metrics) обчислюються лише один раз під час завантаження екрану.
